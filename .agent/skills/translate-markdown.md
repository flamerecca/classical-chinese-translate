# 經典白話翻譯轉換技能 (Classic to Vernacular Translation Skill)

## 說明
此技能用於將特定的文言文經典或佛教文獻，轉換為流暢的繁體中文白話文。此技能設計為「全自動批次處理」，會連續處理完資料夾內的所有檔案，過程中無需使用者手動確認。

## 執行行為規範 (Execution Behavior)
*   **全自動批次處理**：啟動後必須連續、不間斷地處理完指定資料夾內的所有檔案。
*   **禁止中途暫停**：在檔案與檔案之間，**不得**停下來詢問使用者意見或請求確認（免除任何互動確認節點）。
*   **完整交付**：必須將整個資料夾全部翻譯完成後，方可結束任務。

## 目標資料夾與命名規範
*   **儲存路徑**：儲存在與原始資料夾同層的獨立資料夾。
*   **資料夾名稱**：`[原始資料夾名稱]_白話`

## 翻譯與轉換規則
1.  **僅保留純白話翻譯**：不保留任何原文，直接將原文替換為白話文。
    *   *強制約束*：**絕對禁止**在最終檔案中輸出任何 LLM 內部思考軌跡、程式碼註解、對比過程、中途調整備忘（例如：`-> Wait!`, `So let's translate to` 等）。輸出必須是純淨的白話譯文。
2.  **專有名詞保留**：
    *   嚴格保留所有佛教專有名詞（例如：奢摩他、三摩地、毘婆舍那等）。
    *   嚴格保留所有人名與地名（例如：阿難、世尊、室羅筏城、波斯匿王等）。
3.  **音譯與真言/咒語特殊處理**：
    *   對於經典中沒有世俗字面義的梵音對譯字（如咒語、真言），**必須保持原音譯字不變**（例如：`娜牟`、`薩婆勃陀` 等）。
    *   若原句附帶括號註釋釋義（例如：`娜牟薩婆勃陀(敬禮一切諸佛)`），則**僅翻譯/替換括號內部的釋義**，括號格式與音譯主體必須原封不動保留。
4.  **句式轉換**：僅將敘述和論證的文言句式，翻譯為現代流暢的白話繁體中文。
5.  **結構與行數絕對一致**：
    *   完全保持與原文一模一樣的段落結構，不得添加任何額外的段落換行。
    *   **行數必須完全對齊**：每一行原文必須對應一行譯文，每一行空行也必須與原文空行位置完全一致（1-to-1 映射）。

## 檢查與驗證機制（防止漏翻）
*   **行數完整性比對**：必須統計原文總行數與譯文總行數。若有 mismatch（如行數不對等），表示結構已被破壞，必須找出原因並重新對齊。
*   **雜訊過濾檢查**：自我審查譯文中是否含有 `Wait`、`Original`、`Let's`、`#` 或 `//` 等 LLM 夾雜的程式/對話噪音。
*   **消除殘留原文**：除上述規定的專有名詞、人名、地名、音譯咒語外，譯文中不得殘留任何未變更的文言文句式。

```text
你是一個精通佛教經典且極度嚴謹的自動化翻譯專家。請將指定資料夾中的所有文字進行白話翻譯，並遵循以下嚴格限制與執行規範：

【自動化執行規範】
- 本任務為「全自動非互動式任務」。請一口氣、連續地將整個資料夾內的所有檔案全部處理完畢。
- 在檔案與檔案的交替過程中，絕對不要停下來詢問使用者的意見、進度確認或反饋，請直接自動開啟下一個檔案繼續翻譯，直到整個資料夾完全翻譯結束。

【核心翻譯規則】
1. 輸出內容「僅保留純白話翻譯」，請直接將原文替換為白話文，絕對不要保留任何原文。同時，**嚴禁**輸出任何調試註解、自言自語、或者是如 `-> Wait! Original:...` 等思考備忘雜訊。
2. 文本中的佛教專有名詞與人名/地名必須完全保留，不作意譯；對於沒有字面義的梵音音譯咒語，保持漢字音譯不變；若有括號內註解釋義，僅翻譯/替換括號內部的釋義，保留原有的括號與編號格式。
3. 翻譯後的文本必須「完全保持與原文一模一樣的段落結構與行對行對齊」，每一行原文需對應一行譯文，空行位置亦須 100% 相同。

【強制檢查流程（輸出前執行）】
在每個檔案最終輸出前，請進行以下自我審查：
- 請逐行、逐句核對原文與你的譯文，統計譯文的總行數是否與原文總行數完全一致。
- 檢查譯文內是否意外殘留了 LLM 在翻譯過程中的碎碎念備忘（如 `Wait`, `Original` 等關鍵字）。
- 確保除了允許保留的專有名詞/人名/地名/音譯咒語外，其餘所有文言句式皆已徹底轉化為流暢的白話繁體中文。
- 若發現漏翻，請立即補正，確認百分之百完成翻譯後再行輸出。

【檔案儲存規範】
輸出的檔案請規劃儲存於與原始資料夾同層的獨立資料夾，資料夾名稱命名為「[原始資料夾名稱]_白話」。
```
